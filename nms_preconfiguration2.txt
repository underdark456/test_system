Предварительная конфигурация теста сети с использованием имеющейся конфигурации

1. Загрузить конфигурацию в НМС.

2. Получить список требуемых UHP для теста (методы get_uhp(), get_uhp_by_model() get_uhp_controllers_stations())

3. Обновить параметры контроллера(ов): `device_ip`, `device_vlan`, `device_gateway`, `uhp_model`.
	Значения параметров берутся из полученного списка доступных UHP.
    Обновить параметр `tx_level` из опций если требуется.

4. Обновить параметр `stn_vlan` для сервиса, который используется в станционных роутах (IP_address, Static_route).
    Значение параметра берётся из полученного списка UHP.

5. Обновить IP_address станций. Значение берётся из списка UHP.
    Если этого не сделать, станция будет недоступна после UP.

6. Обновить Static_route станций. Значение берётся из списка UHP.
    Если этого не сделать, станция может быть недоступна после UP.

7. Обновить серийные номера станций. Значения берутся из полученного списка UHP.

8. Установить NMS permission для контроллера(ов):
    UHP драйвер инициализирован для каждого UHP, доступен по ключу `web_driver`.
	метод set_nms_permission(vlan=0, password=’’) UHP драйвера .
	Значения параметров `vlan` берётся из списка UHP, значение параметра `password` - из network

9. Установить требуемые профили на станциях:
	методы star_station(), hubless_station(), dama_station(), mesh_station() UHP драйвера.

10. Получить состояние UP для всех требуемых объектов: контроллер(ы), станции, девайсы:
	методы wait_up(), wait_state() и т.д.

11. Подождать 3 тика, чтобы станции получили конфиг: метод wait_ticks(3)
